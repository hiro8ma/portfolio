Google が Gemini Spark を投入 — 端末を閉じても Google Cloud 上で動き続ける『24/7 常駐エージェント』、Claude Cowork / ChatGPT Agent と Workspace を奪い合う 3 巴へ

googlegeminiagentworkspaceai

概要

2026年5月19日、Google I/O 2026 で Sundar Pichai は Gemini Spark を発表した。Gemini base モデルと Google Antigravity の agentic harness を組み合わせた パーソナル AI エージェント だ。

項目内容
発表Google I/O 2026（2026-05-19）
プロダクト名Gemini Spark
ベースGemini モデル + Google Antigravity agentic harness
動作環境Google Cloud の専用 VM 上、24/7
標準統合Gmail / Docs / Sheets / Slides / Chrome / Android Halo
外部拡張MCP（Model Context Protocol）、Canva / OpenTable / Instacart 等
提供開始Trusted tester 公開（5/19 週）、Google AI Ultra ベータ（US、5/26 週）、Mac app（夏）

「あなたのパーソナル AI エージェント。デジタルライフをあなたの代わりに、あなたの指示の下でナビゲートし、行動する」 — Sundar Pichai, CEO, Alphabet（I/O 2026 基調講演）

ポイントは 「端末を閉じても動き続ける」 こと。これまでの "個人エージェント" はブラウザタブ / CLI / アプリ内に閉じていたが、Spark は VM 上で常駐 する。

Spark のアーキテクチャ

構成要素

レイヤー内容
推論モデルGemini ファミリー（3.5 系含む）
エージェントハーネスGoogle Antigravity（既存の Gemini Enterprise でも使われる）

標準ツール群Chrome（ウェブアクセス）、Gmail / Docs / Sheets / Slides

24/7 常駐の意味

観点従来の "個人 AI"（ブラウザ / アプリ）Gemini Spark
起動条件ユーザーがチャットを開いたとき常時 ON
ノートPC を閉じたら止まる動き続ける
数時間〜数日かかるタスク困難設計対象
課金単位API トークン or 月額サブスク（Google AI Ultra）

VM 上で常駐する設計は、ChatGPT Agent や Claude Cowork も近い構造を持つが、Google Workspace に深く一体化 している点で Spark は独自。

「上司に進捗メールを送る必要がある？ Spark がメール・Docs・Sheets・Slides から事実を集めて下書きしてくれる」 — Josh Woodward, VP, Gemini App and AI Studio

日本への示唆

Workspace ユーザーへの実質的影響

観点国内 Workspace ユーザー
大企業（Google Workspace 利用）サイバーエージェント、メルカリ、楽天、リクルート、DeNA、ZOZO 等
中堅・SMBクラウド移行で Workspace 採用が拡大中
公共・大学一部教育委員会・大学で Workspace

Spark は Google Workspace 加入の延長線で導入可能 な構造になる見込み。Microsoft 365 Copilot を選んでいる日本企業（メガバンク・SIer 主流）と、Google Workspace を選んでいる新興・テック企業の間で 「常駐エージェントの差」が業務体験を直接分ける 状況になる。

Microsoft Copilot Studio との対比

観点Microsoft 365 CopilotGoogle Gemini Spark
統合先Excel / Word / PowerPoint / Outlook / TeamsGmail / Docs / Sheets / Slides
実行モデルデスクトップ / Office タブ内クラウド 24/7 VM
端末閉鎖時の挙動止まる動き続ける
エージェント設計Copilot Studio（プロコード/ローコード）Antigravity ハーネス
価格帯（個人）Copilot Pro $20/月Google AI Ultra（未公開、$249/月 と推定する報道あり）

Microsoft Copilot は タブ内ヘルパー から始まり、エージェント化を進めている。Google Spark は最初から クラウド常駐 で来た。設計思想の差が、5 年先のエンドユーザー体験を分ける。

日本のホワイトカラー業務への波及

業務領域Spark が代替し得る部分
上長への進捗報告メールGmail / Docs / Sheets を横断して下書き
経費精算クレカ明細解析
学校・幼稚園・PTA 連絡受信箱監視と締切抽出
顧客問合せ初動SMB 受信箱 24/7 監視
議事録ドラフトDocs + メール文脈で生成

日本の管理職層が「メールを 1 日 2-3 時間さばく」業務時間は、Spark のような 常駐型タスク代行 で 50% 以上削減される射程に入る。これは Stanford AI Index で議論された「AI の生産性インパクト」の体感版だ。

Google Cloud の戦略的勝ち筋

Google が Workspace × Cloud × Gemini モデル × エージェント をスタックで持つことの意味:

レイヤーGoogle 保有競合
OS / モバイルAndroid HaloMicrosoft（Windows）/ Apple（iOS）
ブラウザChrome—
メール・ドキュメントGmail / Docs / Sheets / SlidesMicrosoft 365
クラウド基盤Google CloudAWS / Azure
モデルGemini ファミリーClaude / GPT
エージェントハーネスAntigravityComputer Use / Claude Cowork / ChatGPT Agent

OpenAI / Anthropic はモデルとエージェントは持つが Workspace と OS を持たない。Microsoft は OS と Workspace を持つが モデルは OpenAI 依存。Google は 唯一フルスタックで自社保有 する。Gemini Spark はこの優位を集約して見せた製品だ。

プレイヤー既存提案Spark 公開後

国内提案の 「Copilot vs Spark」 という 2 択比較が、これから本格化する。
