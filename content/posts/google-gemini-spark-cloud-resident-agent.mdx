## 概要

2026年5月19日、Google I/O 2026 で Sundar Pichai は **Gemini Spark** を発表した。Gemini base モデルと **Google Antigravity** の agentic harness を組み合わせた **パーソナル AI エージェント** だ。

| 項目 | 内容 |
|------|------|
| 発表 | Google I/O 2026（2026-05-19） |
| プロダクト名 | Gemini Spark |
| ベース | Gemini モデル + Google Antigravity agentic harness |
| 動作環境 | Google Cloud の専用 VM 上、24/7 |
| 標準統合 | Gmail / Docs / Sheets / Slides / Chrome / Android Halo |
| 外部拡張 | MCP（Model Context Protocol）、Canva / OpenTable / Instacart 等 |
| 提供開始 | Trusted tester 公開（5/19 週）、Google AI Ultra ベータ（US、5/26 週）、Mac app（夏） |

> 「あなたのパーソナル AI エージェント。デジタルライフをあなたの代わりに、あなたの指示の下でナビゲートし、行動する」
> — Sundar Pichai, CEO, Alphabet（I/O 2026 基調講演）

ポイントは **「端末を閉じても動き続ける」** こと。これまでの "個人エージェント" はブラウザタブ / CLI / アプリ内に閉じていたが、**Spark は VM 上で常駐** する。

## Spark のアーキテクチャ

### 構成要素

| レイヤー | 内容 |
|---------|------|
| 推論モデル | Gemini ファミリー（3.5 系含む） |
| エージェントハーネス | Google Antigravity（既存の Gemini Enterprise でも使われる） |
| 実行基盤 | Google Cloud の専用 VM（ユーザー単位） |
| 標準ツール群 | Chrome（ウェブアクセス）、Gmail / Docs / Sheets / Slides |
| モバイル連携 | Android Halo（進捗トラッキング） |
| 外部拡張 | MCP、Canva / OpenTable / Instacart 等 |

### 24/7 常駐の意味

| 観点 | 従来の "個人 AI"（ブラウザ / アプリ） | Gemini Spark |
|------|---------------------------------|------------|
| 起動条件 | ユーザーがチャットを開いたとき | 常時 ON |
| ノートPC を閉じたら | 止まる | 動き続ける |
| 数時間〜数日かかるタスク | 困難 | 設計対象 |
| 課金単位 | API トークン or 月額 | サブスク（Google AI Ultra） |

VM 上で常駐する設計は、ChatGPT Agent や Claude Cowork も近い構造を持つが、**Google Workspace に深く一体化** している点で Spark は独自。

## 想定されたユースケース

VP の Josh Woodward が示した具体例:

| ユースケース | 動作 |
|------------|------|
| 月次クレカ明細の解析 | 隠れたサブスクや手数料を抽出して報告 |
| 子供の学校からの通知監視 | Gmail を監視、締切や提出物を抽出して daily report |
| 上司への進捗メール起草 | Gmail / Docs / Sheets / Slides から事実を集めて下書き |
| スモールビジネスの問い合わせ管理 | 受信箱を 24/7 監視、顧客問合せを取りこぼさない |

> 「上司に進捗メールを送る必要がある？ Spark がメール・Docs・Sheets・Slides から事実を集めて下書きしてくれる」
> — Josh Woodward, VP, Gemini App and AI Studio

## Claude Cowork / ChatGPT Agent との 3 巴

3 ベンダーの "個人 AI エージェント" を整理する。

| 観点 | Gemini Spark | Claude Cowork | ChatGPT Agent |
|------|------------|--------------|--------------|
| 提供元 | Google | Anthropic | OpenAI |
| 実行環境 | Google Cloud 専用 VM、24/7 | Anthropic 基盤 | OpenAI 基盤 |
| Workspace 統合 | **Gmail / Docs / Sheets / Slides に out-of-the-box** | Cowork ドキュメント中心 | ChatGPT 内 |
| 外部接続 | MCP + Canva / OpenTable / Instacart | MCP | Browser / カスタム接続 |
| モバイル連携 | Android Halo | iOS / Android Claude アプリ | ChatGPT アプリ |
| 課金 | Google AI Ultra（サブスク） | Claude Max / Cowork プラン | ChatGPT Plus / Pro |
| 主要強み | Workspace の深い統合 | 推論精度・倫理境界 | 配布力（週次 800M+ ユーザー） |

**Google の構造的優位は Gmail / Docs / Sheets / Slides** だ。米国・欧州・日本のホワイトカラーの大多数が日常で使うインフラを、認証・権限を経由せず Spark が直接読み書きする。

## 日本への示唆

### Workspace ユーザーへの実質的影響

| 観点 | 国内 Workspace ユーザー |
|------|----------------------|
| 大企業（Google Workspace 利用） | サイバーエージェント、メルカリ、楽天、リクルート、DeNA、ZOZO 等 |
| 中堅・SMB | クラウド移行で Workspace 採用が拡大中 |
| 公共・大学 | 一部教育委員会・大学で Workspace |

Spark は **Google Workspace 加入の延長線で導入可能** な構造になる見込み。Microsoft 365 Copilot を選んでいる日本企業（メガバンク・SIer 主流）と、Google Workspace を選んでいる新興・テック企業の間で **「常駐エージェントの差」が業務体験を直接分ける** 状況になる。

### Microsoft Copilot Studio との対比

| 観点 | Microsoft 365 Copilot | Google Gemini Spark |
|------|----------------------|--------------------|
| 統合先 | Excel / Word / PowerPoint / Outlook / Teams | Gmail / Docs / Sheets / Slides |
| 実行モデル | デスクトップ / Office タブ内 | クラウド 24/7 VM |
| 端末閉鎖時の挙動 | 止まる | 動き続ける |
| エージェント設計 | Copilot Studio（プロコード/ローコード） | Antigravity ハーネス |
| 価格帯（個人） | Copilot Pro $20/月 | Google AI Ultra（未公開、$249/月 と推定する報道あり） |

Microsoft Copilot は **タブ内ヘルパー** から始まり、エージェント化を進めている。Google Spark は最初から **クラウド常駐** で来た。設計思想の差が、5 年先のエンドユーザー体験を分ける。

### 日本のホワイトカラー業務への波及

| 業務領域 | Spark が代替し得る部分 |
|---------|-------------------|
| 上長への進捗報告メール | Gmail / Docs / Sheets を横断して下書き |
| 経費精算 | クレカ明細解析 |
| 学校・幼稚園・PTA 連絡 | 受信箱監視と締切抽出 |
| 顧客問合せ初動 | SMB 受信箱 24/7 監視 |
| 議事録ドラフト | Docs + メール文脈で生成 |

日本の管理職層が「メールを 1 日 2-3 時間さばく」業務時間は、Spark のような **常駐型タスク代行** で 50% 以上削減される射程に入る。これは Stanford AI Index で議論された「AI の生産性インパクト」の体感版だ。

### Google Cloud の戦略的勝ち筋

Google が **Workspace × Cloud × Gemini モデル × エージェント** をスタックで持つことの意味:

| レイヤー | Google 保有 | 競合 |
|---------|------------|------|
| OS / モバイル | Android Halo | Microsoft（Windows）/ Apple（iOS） |
| ブラウザ | Chrome | — |
| メール・ドキュメント | Gmail / Docs / Sheets / Slides | Microsoft 365 |
| クラウド基盤 | Google Cloud | AWS / Azure |
| モデル | Gemini ファミリー | Claude / GPT |
| エージェントハーネス | Antigravity | Computer Use / Claude Cowork / ChatGPT Agent |

OpenAI / Anthropic はモデルとエージェントは持つが **Workspace と OS を持たない**。Microsoft は OS と Workspace を持つが **モデルは OpenAI 依存**。Google は **唯一フルスタックで自社保有** する。Gemini Spark はこの優位を集約して見せた製品だ。

### 国内 SIer / コンサルの提案戦略

| プレイヤー | 既存提案 | Spark 公開後 |
|----------|---------|-------------|
| NEC / 富士通 / NTT データ | Microsoft 365 Copilot を軸 | Google Workspace 顧客向けに Spark 統合を並列提案 |
| アクセンチュア / DTC | Copilot Studio + AOAI ベースの業務エージェント | Antigravity ハーネスを使う Google パスの提案を整備 |
| GCP パートナー（CIS / G-gen 等） | データ基盤 + Vertex AI | Spark 接続のカスタム MCP コネクタ構築 |

国内提案の **「Copilot vs Spark」** という 2 択比較が、これから本格化する。
