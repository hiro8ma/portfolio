## 概要

2026年5月の同じ週、Anthropic と OpenAI が **AI ネイティブ・エンタープライズサービス JV** を相次いで発表した。

| 日付 | 企業 | JV 名 | 評価額 | 主要パートナー |
|------|------|------|-------|------------|
| 5月4日 | **OpenAI** | The Development Company | $10B | TPG、Brookfield、Advent、Bain Capital（19 投資家、$4B 調達） |
| 5月5日 | **Anthropic** | Wall Street JV | $1.5B | Blackstone、Hellman & Friedman、Goldman Sachs、Apollo、General Atlantic、Leonard Green、GIC、Sequoia |

両社の戦略は同じ — **Asset Manager と組んでポートフォリオ企業に preferred sales access** を提供し、forward-deployed engineer モデルで導入を加速する。

Anthropic 側は Claude Opus 4.7 を金融特化モデルとして披露し、JPMorgan Chase・Goldman Sachs・Citi・AIG・Visa で **既に本番稼働** していることを明らかにした。

> 「20分で巨大なダッシュボードと裏付け・調査をすべて作り上げた。極めて正確だった」
> — Jamie Dimon, JPMorgan Chase CEO

> 「Anthropic は 10倍成長を予想したが、四半期で 80倍 になった」
> — Dario Amodei, Anthropic CEO

### Pre-built Finance Agents

10 種類のエージェントが reference architecture として配布される。

| カテゴリ | エージェント例 |
|---------|------------- |
| 投資銀行業務 | Pitchbook 作成、Earnings 分析 |
| 与信・引受 | Credit Memo、Underwriting |
| コンプライアンス | KYC、Statement Audit |
| 経理 | Month-end Close |
| 保険 | Insurance Claims（AIG 88% 一致率） |

### データ連携

| 種類 | パートナー |
|------|-----------|
| 企業信用 | Moody's（600M+ 社の credit rating を Claude に直接埋込） |

| 投資情報 | S&P Capital IQ、Morningstar、PitchBook、LSEG |
| 専門家ネットワーク | GLG、Guidepoint、Third Bridge、Fiscal AI |

### Microsoft 365 統合

Claude が Excel・PowerPoint・Word に **Add-in** として統合され、Outlook はベータ。Excel での財務モデリング・PowerPoint での pitchbook 自動化が直接できる構造になった。

これは Microsoft が OpenAI に依存している中での **クロス契約** で、Microsoft 365 ユーザーが Claude を選べる選択肢を持つ意味は大きい。

OpenAI 側は具体的な業界縦割り（vertical）を明示せず、Anthropic の Wall Street 集中とは違う **横断的なエンタープライズ展開** を狙う。

OpenAI は別途 $122B の調達発表もしており、Anthropic は $900B 評価で $50B 調達を進める。両社は **資本調達と JV 展開を同時並行** で進める段階に入った。

## なぜ同じ週に？

### 共通する戦略構造

| 要素 | 内容 |
|------|------|
| Asset Manager 出資 | ポートフォリオ企業への preferred access を確保 |
| Forward-deployed engineer | コンサル + エンジニアが顧客現場に常駐して導入 |
| Pre-built agents | 業界別 reference architecture を配布 |
| データ連携 | 業界の主要データプロバイダーを embed |

両社のモデルは Anthropic の表現で言えば「**エンジニアリングチームが顧客の臨床医・IT スタッフと座って業務に組み込むツールを構築する**」スタイル。これは伝統的な SI でなく、コンサル + AI ベンダーの混血だ。

### 競争の本質

| 観点 | Anthropic | OpenAI |
|------|-----------|--------|
| 業界フォーカス | 金融・コンプライアンス・保険 | 業界横断 |
| 大手バンク本番稼働 | JPMorgan、Goldman、Citi、AIG、Visa | 未公表 |
| Microsoft 365 | 統合（add-in 完成） | Microsoft 経由（Copilot） |
| 成長率 | 80x（年換算）/四半期 | 非公表 |
| パートナー資本 | $1.5B JV | $4B JV |
| 投資家重複 | — | Anthropic と 0 重複 |

## 日本への示唆

### 日本のメガバンクは次のターゲット

Anthropic は Wall Street で JPMorgan・Goldman・Citi を押さえた。次は **欧州大手・アジア大手** が照準だ。日本では:

| 銀行 | AI 戦略の現状 |
|------|------------|
| MUFG | 国産 LLM + Microsoft Copilot を併用 |
| SMBC | 自社 AI 基盤 + Microsoft 365 |
| Mizuho | みずほ × NTT データ AI 連携、生成 AI 領域は外部依存 |
| Resona | 国産モデル中心 |

NEC と Anthropic の戦略提携には金融業界向けが含まれているが、Wall Street JV ほどの **Pre-built finance agent** はまだ日本に上陸していない。Wall Street で JPMorgan が 20分で pitchbook を作る世界が、日本のメガバンクで再現されるのは時間の問題だ。

### コンサル・SIer への波及

| 既存プレイヤー | 想定影響 |
|------------ |---------|
| 大手金融コンサル（McKinsey、BCG、Bain 等の Japan 部門） | Anthropic forward-deployed engineer と直接競合 |
| 国内 SIer（NRI、TIS、SCSK 等） | reference architecture 配布で工数ビジネスが圧迫 |
| Big4 コンサル | Microsoft 365 add-in × Claude で監査・KYC ワークが自動化 |

「Pre-built agent + データ連携」を提供できるコンサル・SIer と、「人月で受託する」だけの会社が分岐する。Anthropic の 80x 四半期成長は、業界が agent ベース調達に一気に動いた指標として読める。

### Microsoft 365 add-in の意味

Microsoft Copilot vs Claude add-in は **同じ Excel/PowerPoint の中で切替可能** になる。日本企業の業務は Microsoft 365 が標準で、ここに Claude が直接入ることで「ChatGPT を別タブで開く」ステップが消える。

これは情シス調達の前提を変える。「Copilot を全社標準にする」決定を保留して **Claude add-in も併用評価する** 動きが出てくる。

## まとめ

- 2026年5月4日に OpenAI が The Development Company（$10B 評価、$4B 調達）、5月5日に Anthropic が Wall Street JV（$1.5B、Blackstone・Goldman 等）を相次いで発表
- Anthropic は Claude Opus 4.7（Vals AI Finance Agent 64.4% 首位）を JPMorgan・Goldman・Citi・AIG・Visa で **既に本番稼働**。Moody's の credit rating + 主要金融データを直接 embed
- 共通する戦略: Asset Manager 出資による preferred sales access、forward-deployed engineer モデル、pre-built agent 配布、業界データ連携
- Microsoft 365 add-in（Excel/PowerPoint/Word/Outlook）対応で Microsoft Copilot と直接競合する形に
- 日本のメガバンクと国内コンサル・SIer は次の波の影響を受ける。「Pre-built agent + データ連携」を提供できる側と工数受託のみの側で分岐が始まる
