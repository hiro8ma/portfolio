# Claude Code リリースノート分析 & 投稿ガイド

## 1. リリースノートの取得方法

### CHANGELOG の場所

```bash
# GitHub から直接取得
curl -s https://raw.githubusercontent.com/anthropics/claude-code/main/CHANGELOG.md | head -200

# npm でバージョン確認
npm view @anthropic-ai/claude-code version
```

### 確認すべきソース

| ソース | URL | 用途 |
|--------|-----|------|
| **CHANGELOG.md** | `github.com/anthropics/claude-code/blob/main/CHANGELOG.md` | 公式変更履歴 |
| **GitHub Releases** | `github.com/anthropics/claude-code/releases` | リリースノート |
| **npm** | `npmjs.com/package/@anthropic-ai/claude-code` | バージョン情報 |
| **X/Twitter** | `@anthropikiryoku`、`@thariqshihipar` など | 公式発表 |

---

## 2. CHANGELOG の読み方

### セクション構造

```markdown
## [バージョン番号] - 日付

### Added（新機能）
### Changed（変更）
### Fixed（バグ修正）
### Deprecated（非推奨）
### Removed（削除）
### Security（セキュリティ）
```

### 注目すべきポイント

| 優先度 | カテゴリ | 理由 |
|--------|----------|------|
| **高** | 新コマンド/機能 | ユーザーの使い方が変わる |
| **高** | 破壊的変更 | 既存ユーザーに影響 |
| **中** | バグ修正（重大） | 多くのユーザーが遭遇していた問題 |
| **中** | パフォーマンス改善 | 体感に影響 |
| **低** | 内部リファクタリング | 一般ユーザーには見えない |
| **低** | マイナーなバグ修正 | 影響範囲が限定的 |

## 3. ブログ記事の書き方

### 記事構成テンプレート

```markdown
---
title: "Claude Code {version} リリース — {目玉機能}"
date: "{YYYY-MM-DD}"
description: "{1〜2文の要約}"
tags: ["claude-code", "anthropic", "ai", "cli", "release"]
---

## Claude Code {version} がリリース
[1段落で概要]

## 新機能
### {機能名} — {一言説明}
[コード例]
[ユースケース]

## バグ修正
### {問題の説明}
**問題**: ...
**修正済み**。

## 改善点
[箇条書きまたはテーブル]

## アップデート方法
```bash
npm update -g @anthropic-ai/claude-code
```

## まとめ
[テーブルで要約]

## 参考リンク
- [Claude Code GitHub](...)
```

### 書き方のポイント

| ポイント | 説明 |
|----------|------|
| **見出しで完結** | `### /loop コマンド — 定期プロンプト実行` |
| **コード例は必須** | 実際に動くコマンドを示す |
| **テーブルを多用** | 情報を整理して見やすく |
| **短い段落** | 1段落は2〜3文まで |
| **日本語 + 英語用語** | 専門用語は英語のまま |

### 避けるべきこと

- 長い導入文（すぐ本題へ）
- 「〜と思います」「〜かもしれません」（断定調で）
- 全部の変更を網羅（重要なものを厳選）
- 公式の文言をそのまま翻訳（自分の言葉で解釈）

## 4. X（Twitter）投稿の書き方

### 基本フォーマット

```
Claude Code v{version} がリリース {絵文字}

{目玉機能を1行で}

主な変更:
• {変更1}
• {変更2}
• {変更3}

#ClaudeCode #AI
```

### 文字数ガイド

| 要素 | 目安 |
|------|------|
| **全体** | 140文字以内（画像なし）、280文字以内（画像あり） |
| **ハッシュタグ** | 2〜3個まで |
| **改行** | 読みやすさのため積極的に使う |

### 絵文字の使い分け

| 絵文字 | 用途 |
|--------|------|
| 🎉 | 新機能、メジャーリリース |
| 🔧 | バグ修正、改善 |
| ⚡ | パフォーマンス改善 |
| 🚀 | 大きな新機能 |
| 🔊 | Voice 関連 |
| 🛡️ | セキュリティ |

### 投稿例

#### 新機能メイン

```
Claude Code v2.1.71 リリース 🎉

/loop コマンドで定期実行が可能に！
デプロイ監視やテスト実行を自動化

• Voice キーバインドカスタマイズ
• stdin フリーズ修正
• 起動時間短縮

#ClaudeCode #AI
```

#### バグ修正メイン

```
Claude Code v2.1.71 🔧

安定性が大幅向上:
• stdin フリーズ問題を修正
• Voice 起動時のフリーズ解消
• OAuth トークン更新の改善

長時間セッションがより快適に

#ClaudeCode
```

### NG パターン

| NG | 理由 |
|----|------|
| 「〜がリリースされました！！」| 感嘆符過剰 |
| 変更を全部列挙 | 読めない |
| リンクだけ | 何が変わったか分からない |
| 英語の CHANGELOG をそのまま | 日本語で書く |

## 5. 判断基準

### 投稿する価値があるか

| 条件 | 判断 |
|------|------|
| 新コマンド追加 | **投稿する** |
| 重大なバグ修正 | **投稿する** |
| 破壊的変更 | **投稿する**（警告として） |
| マイナーバグ修正のみ | スキップ |
| 内部リファクタリングのみ | スキップ |

### ブログ vs X のみ

| 変更の規模 | 対応 |
|------------|------|
| 新機能 + 複数の改善 | ブログ + X |
| 1つの目玉機能 | X のみ（または短いブログ） |
| バグ修正のみ | X のみ |
| マイナー | スキップ |

## 6. 自動チェックの設定

### Claude スケジュールタスク用プロンプト

```
Claude Code の最新バージョンをチェック:

1. npm view @anthropic-ai/claude-code version
2. ~/.claude-code-version と比較
3. 新バージョンなら:
   - CHANGELOG.md を取得
   - 上記ガイドに従って分析
   - X投稿ドラフトを作成
   - Mac デスクトップに通知
```

### 通知内容

新バージョン検出時に以下を出力:
1. バージョン番号
2. 主要な変更点（日本語要約）
3. X投稿用ドラフト
4. ブログを書く価値があるかの判断
